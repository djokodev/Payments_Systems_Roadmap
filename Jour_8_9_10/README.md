`Jours 8-9-10 : Programmation orientée objet`

Ces jours étaient consacrés à la POO en Python. Modéliser le monde
réel sous forme d'objets qui ont des données et des comportements.

Concept clé : une classe est un moule, un objet est ce qu'on fabrique
avec. Chaque wallet est un objet distinct avec ses propres données.

Concepts pratiqués :
- __init__, attributs d'instance
- @property pour protéger l'accès au solde
- Méthodes métier : credit(), debit(), get_daily_limit()
- __str__, __repr__, __eq__

Exercice principal : modéliser un Wallet mobile money camerounais avec
ses vraies contraintes métier, solde protégé, vérification KYC,
blocage de compte, limites journalières.

Quelques exercices supplémentaires de pratique sur les structures
de données (dict, list, comprehensions).
